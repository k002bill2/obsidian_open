# 클로드 코드: 비동기 서브에이전트와 병렬 워크플로우

앤스로픽(Anthropic)의 **Claude Code**에 새롭게 도입된 **비동기 백그라운드 서브에이전트** 기능과 기타 업데이트 사항에 대해 상세히 정리해 드립니다.

## 1. 비동기 백그라운드 서브에이전트 (Asynchronous Background Sub-agents)

이번 업데이트의 핵심은 **진정한 병렬 작업의 구현**입니다. 이전에는 여러 작업을 수행하기 위해 여러 터미널 인스턴스를 열어야 했으나, 이제는 Claude Code 내부적으로 이를 관리할 수 있게 되었습니다.
• **작동 방식:** 메인 에이전트에게 서브에이전트 활용이 필요한 작업을 요청하면 실행이 시작됩니다. 이때 **`Control+B`**를 눌러 해당 작업을 백그라운드로 보내고, 메인 에이전트로는 다른 작업을 계속 진행할 수 있습니다.
• **자율성:** 서브에이전트는 별도의 프로세스로 작동하며 복잡한 다단계 작업을 독립적으로 처리합니다. 작업이 완료되면 그 결과를 가지고 메인 에이전트를 다시 활성화하여 보고합니다.
• **상태 확인:** 현재 실행 중인 서브에이전트의 목록은 **tasks** 명령어로 확인할 수 있습니다.

## 2. 주요 활용 사례 및 장점

서브에이전트는 특히 **메인 컨텍스트 창을 깨끗하게 유지**해야 하거나 시간이 오래 걸리는 작업에 유용합니다.
• **병렬 연구 및 검토:** 시간이 많이 걸리는 연구 작업, 보안 검토, 성능 병목 현상 식별, 코드 리팩토링 등을 백그라운드에서 동시에 수행할 수 있습니다.
• **동시 대규모 수정:** 여러 페이지의 변경 사항을 동시에 적용하거나, 모든 컴포넌트를 새로운 라이브러리로 전환하는 작업을 한 번에 처리할 수 있습니다.
• **Git 워크트리(Worktree) 활용:** 가장 혁신적인 활용법 중 하나로, 서브에이전트에게 별도의 Git 워크트리를 할당하여 **여러 브랜치에서 독립적으로 커밋과 푸시를 수행**하도록 할 수 있습니다. 예를 들어, 세 가지 다른 테마를 병렬로 구현해 본 뒤 마음에 드는 것만 메인 프로젝트에 병합하는 식의 실험이 가능합니다.

## 3. 기타 주요 업데이트 기능

• **즉시 자동 압축:** 프로젝트 기록을 Claude 폴더에 직접 유지함으로써 이전보다 훨씬 빠르게 세션 기록을 압축합니다.
• **프롬프트 제안:** 엔터 키로 간단히 수락할 수 있는 프롬프트 제안 기능이 추가되었습니다.
• **세션 포크(Forking):** **Esc****키를 두 번** 누르면 현재 세션을 포크할 수 있으며, `resume` 플래그를 통해 다시 시작할 수 있습니다.
• **에이전트 플래그:** 실행 시 `agent` 플래그를 사용하여 프로젝트 내 특정 에이전트에게 작업을 직접 위임할 수 있습니다.

## 4. 주의사항 및 모범 사례

• **사용자 개입 금지:** 사용자 입력이나 승인이 필요한 작업은 백그라운드로 보내지 않아야 합니다. 에이전트가 중단되거나 메인 에이전트가 바쁠 때 상태를 놓칠 수 있기 때문입니다.
• **독립적 작업 할당:** 서로 의존성이 있는 작업은 충돌을 일으킬 수 있으므로, 각 에이전트에게는 독립적인 작업을 할당하는 것이 권장됩니다.
• **토큰 관리:** 서브에이전트는 속도를 높여주지만 **상당한 수의 토큰을 소비**하므로 비용 관리를 위해 사용량을 추적해야 합니다.
• **명확한 명명:** 여러 에이전트를 관리할 때는 설명적인 이름을 지정하여 각 작업의 진행 상황을 정확히 파악하는 것이 좋습니다.

이 새로운 기능들은 마치 **한 명의 개발자가 여러 명의 유능한 조수를 거느리고 각기 다른 방에서 동시에 작업을 시키는 것**과 같습니다. 메인 개발자는 거실에서 전체적인 설계를 유지하면서, 조수들이 각자 방에서 연구나 코드 수정을 완료하고 돌아오기를 기다려 효율적으로 업무를 통합할 수 있게 된 것입니다.
